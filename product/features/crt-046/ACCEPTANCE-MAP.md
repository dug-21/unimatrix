| AC-09 | `context_briefing` with an empty `goal_clusters` table returns results identical to the pre-crt-046 pure semantic path | test | Integration test: `goal_clusters` table empty; call briefing with a goal; assert output IDs and order equal pure-semantic baseline | PENDING |
| AC-10 | Cluster-derived entry IDs that do not have `status = Active` are excluded from briefing results | test | Integration test: seed a `goal_clusters` row whose `entry_ids_json` includes a deprecated entry ID and a quarantined entry ID; call briefing with matching goal; assert neither entry appears in output; assert an Active entry from the same row does appear | PENDING |
| AC-11 | The `goal_clusters` query at briefing time is constrained to the last 100 rows by `created_at` descending; rows outside this window are not scanned | test | Integration test: seed 101 rows in `goal_clusters` where the 101st (oldest by `created_at`) has the highest cosine similarity to the test goal embedding; assert that the oldest row's entry IDs do NOT appear in briefing output (recency cap excludes it) | PENDING |
| AC-12 | The v21→v22 migration creates `goal_clusters` table and its index on a live database without error; all 9 cascade touchpoints from entry #3894 are addressed | test | Migration integration test: open a v21 fixture DB, run migration, assert `read_schema_version(&store) == 22`, assert `goal_clusters` table exists with exactly 7 columns, assert `idx_goal_clusters_created_at` index exists | PENDING |
| AC-13 | When one or more `context_get` observation rows have unparseable `input` JSON during `context_cycle_review`, the returned MCP response includes a non-zero `parse_failure_count` top-level field (outside `CycleReviewRecord`); valid rows still produce edges | test | Integration test: seed one malformed observation row (missing `id` field) alongside two valid rows; call review; assert returned JSON has top-level `parse_failure_count >= 1`; flush drain; assert valid rows produced behavioral edges | PENDING |
