# Scope Risk Assessment: crt-027

## Technology Risks

| Risk ID | Risk | Severity | Likelihood | Recommendation |
|---------|------|----------|------------|----------------|
| SR-01 | SubagentStart stdout injection is unverified at scope time — SCOPE.md assumes Claude Code reads SubagentStart hook stdout and injects it into the subagent context before its first token, but no ADR or ASS spike confirms this behavior. If Claude Code treats SubagentStart as output-ignored (like Stop), the entire WA-4a value proposition fails silently. | High | Med | Architect must confirm or deny stdout injection support for SubagentStart before designing the response path. If unconfirmed, add a spike or fallback design (e.g., passthrough to session state only). |
| SR-02 | `mcp-briefing` feature flag is `default = ["mcp-briefing"]` — new index implementation replaces `BriefingService` inside the same `#[cfg(feature = "mcp-briefing")]` guard. Risk that tests for the new index path are also placed inside `#[cfg(feature = "mcp-briefing")]` blocks, leaving the non-briefing build path untested. The existing `briefing.rs` tests are not flag-gated (briefing.rs is always compiled), so deletion of that file removes ~2,000 lines of tests that ran unconditionally. | Med | Med | Architect must specify whether `IndexBriefingService` lives inside or outside the flag guard, and whether tests require the flag. Ensure `cargo test` without features exercises at minimum the CompactPayload path. |

## Scope Boundary Risks

| Risk ID | Risk | Severity | Likelihood | Recommendation |
|---------|------|----------|------------|----------------|
| SR-03 | `BriefingService` deletion removes `EffectivenessStateHandle` dependency — `BriefingService` holds an `Arc<Mutex<EffectivenessSnapshot>>` cloned from `ServiceLayer`. Removing `BriefingService` without confirming the new `IndexBriefingService` re-establishes this dependency risks breaking effectiveness-based ranking in the new briefing path. `services/mod.rs` wires this Arc into `ServiceLayer`; if the new service doesn't receive it, ranking silently degrades. | High | Med | Spec must explicitly state whether `IndexBriefingService` receives `EffectivenessStateHandle` and how it participates in the effectiveness snapshot pattern (ADR-001 crt-018b). |
| SR-04 | `format_compaction_payload` has 10+ unit tests asserting on `CompactionCategories` structure (decisions / injections / conventions fields). These tests must be rewritten, not deleted — they cover budget enforcement, UTF-8 handling, histogram rendering, and token limits. Scope says to remove section structure but the underlying invariants (budget caps, sorted output, histogram block) are still required. Test loss without replacement is an AC-15 violation risk. | High | High | Spec must enumerate which `format_compaction_payload` test invariants survive the format change and must be covered by new tests. Architect should not assume old tests can be simply deleted. |
| SR-05 | `UNIMATRIX_BRIEFING_K` env var has undefined fate — SCOPE.md notes it "should be documented or superseded — not silently inherited" but does not resolve this. The old default is 3 (clamped [1,20]); new default is 20. If the env var is silently inherited and set to 3 in a deployment, the new index path returns only 3 entries instead of 20, defeating the high-k design. | Med | Med | Spec must explicitly state: env var is deprecated and ignored, OR honored with a floor of 20, OR replaced by a new var. Pick one; document in AC-07. |
| SR-06 | WA-5 format contract is load-bearing but under-specified — SCOPE.md states WA-5 "reads the flat table directly" and "prepend without parsing section structure," but the flat table format (column widths, separator character, row number field width) is defined only by prose example. If the exact byte format shifts during implementation, WA-5 will need rework. | Med | Low | Architect should define the flat table format as a typed struct or a documented constant rather than inline string formatting, so WA-5 can depend on it by contract not by example. |

## Integration Risks

| Risk ID | Risk | Severity | Likelihood | Recommendation |
|---------|------|----------|------------|----------------|
| SR-07 | `HookRequest::ContextSearch` wire protocol addition — adding optional `source` field with `#[serde(default)]` is backward-compatible for deserialization, but any existing serialized/persisted `HookRequest` values (event queue replay, integration test fixtures using literal JSON) that omit `source` will deserialize to the default. Risk is low for IPC path but medium for any test fixture that hard-codes the JSON variant and then does a round-trip assertion on the struct. | Low | Low | Architect confirms `#[serde(default)]` is sufficient. Spec should note that any test constructing `ContextSearch` via struct literal must add `source: None` or `..` pattern to compile after the field addition. |
| SR-08 | Cold-state query derivation — step 2 of the three-step fallback (synthesize from `feature_cycle` + top 3 `topic_signals`) requires `SessionRegistry` lookup via `session_id`. At session start or when `session_id` is absent, `topic_signals` will be empty (WA-1/WA-2 not yet populated). Step 3 fallback to `topic` param (e.g., `"crt-027"`) could produce a low-signal topic-string query that returns only entries whose topic field exactly or semantically matches that string. If the knowledge base has no entries tagged with that topic, k=20 returns zero results. The SCOPE.md guard (`if no session state or empty topic_signals`) does handle this but the fallback quality is unvalidated. | Med | Med | Architect should validate step 3 fallback behavior with an empty-topic query against the live knowledge base; consider whether `topic` param should default to a broader query (e.g., feature phase name) if the feature ID returns zero results. |
| SR-09 | SM protocol update adds `context_briefing` calls at 6 phase boundaries — the briefing response (k=20, 150-char snippets, flat table) consumes context window tokens in the SM's session. No accounting is specified. If the SM is near its context limit at a late phase boundary (e.g., pr-review), the briefing call may be truncated or the SM may lose earlier context. This interacts with WA-5 (PreCompact) but WA-5 is not yet implemented. | Med | Med | Spec should define a max_tokens budget for SM briefing calls (e.g., cap at 1000 tokens) and document this in the protocol update step. The architect should confirm that the SM agent's context budget accounts for briefing overhead. |

## Assumptions

- **SCOPE.md §WA-4a, "The hook response for SubagentStart will be `HookResponse::Entries`... Claude Code reads stdout from the SubagentStart hook and injects it into the subagent context"** — This is an untested assertion about Claude Code host behavior. No ASS spike is cited. SR-01 follows directly.
- **SCOPE.md §Additional finding (empty prompt_snippet handling)** — Assumes empty `query` in `build_request` is already guarded. Confirmed by code inspection: the guard exists at hook.rs lines 255-257. No risk.
- **SCOPE.md §Q4, "Confirmed two callers only"** — Confirmed by code inspection (`tools.rs` line ~942, `listener.rs` line ~1209). `BriefingService` re-exported from `services/mod.rs` and referenced in `services/effectiveness.rs` comments. SR-03 arises from the `EffectivenessStateHandle` dependency that must migrate.
- **SCOPE.md §Design Decision OQ-4, "Remove in this feature"** — Confirmed scope boundary. The risk is not "should we remove it" but "do we replace all its cross-cutting wiring" (SR-03).

## Design Recommendations

- **SR-01 (CRITICAL):** Before architecture begins, verify Claude Code SubagentStart hook stdout behavior via a 30-minute spike (or cite existing documentation). If stdout is not injected, the WA-4a design must pivot to session-state-only recording with no response content.
- **SR-03 + SR-04 (HIGH):** Architect must define `IndexBriefingService` construction to include `EffectivenessStateHandle` (matching `BriefingService` pattern), and spec must enumerate surviving test invariants for `format_compaction_payload` by name so they are rewritten not dropped.
- **SR-08 + SR-09:** Spec should define minimum query string requirements for cold-state fallback and cap the SM briefing token budget at each phase boundary call.
