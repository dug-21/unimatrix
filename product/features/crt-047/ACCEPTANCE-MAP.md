# crt-047 Acceptance Criteria Map

Resolutions applied:
- **FAIL-01**: AC-04 superseded — orphan attribution uses ENTRIES-only query per ADR-003,
  not AUDIT_LOG join as written in SCOPE.md / SPEC AC-04.

  included per ADR-001/ADR-002.
- **WARN-01**: `corrections_system` is a stored column; AC-03 test must assert all four
  buckets.

## Notes

- **AC-04 governing approach**: SCOPE.md AC-04 and SPEC AC-04 both describe an AUDIT_LOG
  join. This is superseded by ADR-003 (ENTRIES-only). The acceptance criterion above reflects
  the ADR-003 resolution. Implementors must not follow the SCOPE/SPEC AC-04 text for the SQL
  design.

- **AC-01 / AC-14 column count**: SCOPE.md AC-01 and SPEC FR-08 list five columns. The
  authoritative count is seven (five snapshot + `corrections_system` per ADR-002 +
  `first_computed_at` per ADR-001). Tests must assert all seven.

- **AC-R01 through AC-R05**: These are additional acceptance criteria derived from ADR-001
  (ordering key stability) and the RISK-TEST-STRATEGY Critical/High risks (R-02, R-07, R-10,
  R-11). They are not in SCOPE.md but are required by the architecture.
