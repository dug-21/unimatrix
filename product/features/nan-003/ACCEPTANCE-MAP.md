# nan-003 Acceptance Criteria Map

| AC-ID | Description | Verification Method | Verification Detail | Status |
|-------|-------------|--------------------|--------------------|--------|
| AC-01 | `/unimatrix-init` appends a Unimatrix block to `CLAUDE.md` containing: (a) `unimatrix-*` skills table with one-line descriptions, (b) category convention guide (decision/pattern/procedure/convention/lesson-learned), (c) usage trigger instructions | manual | Run `/unimatrix-init` on a repo without an existing block; open CLAUDE.md and verify: (a) skills table lists `/unimatrix-init` and `/unimatrix-seed` with descriptions, (b) category guide present with ≥5 categories, (c) when-to-invoke section present | PENDING |
| AC-02 | `/unimatrix-init` is idempotent — running a second time when sentinel is present produces no changes to `CLAUDE.md` and prints "already initialized" | manual | Run `/unimatrix-init` once; capture CLAUDE.md checksum; run again; verify checksum unchanged and terminal prints "already initialized" | PENDING |
| AC-03 | `/unimatrix-init` creates `CLAUDE.md` when the file does not exist | manual | Delete (or confirm absence of) CLAUDE.md; run `/unimatrix-init`; verify CLAUDE.md created containing the Unimatrix block with sentinel | PENDING |
| AC-04 | `/unimatrix-init` scans `.claude/agents/**/*.md` and produces a terminal-only recommendation report with concrete `unimatrix-*` skill examples; no agent files are modified | manual | Run `/unimatrix-init` on a repo with agent files; verify terminal output contains recommendation table; verify agent file timestamps unchanged after run | PENDING |
| AC-05 | `/unimatrix-init --dry-run` prints what would be written and the agent recommendation without modifying any files | manual | Run `/unimatrix-init --dry-run`; verify CLAUDE.md unchanged (or absent) after run; verify terminal output includes block content and agent recommendations | PENDING |
| AC-06 | `/unimatrix-seed` Level 0 automatically reads README, manifests, top-level structure without opt-in; proposes 2–4 high-level foundational entries for batch approval; stores approved entries | manual | Run `/unimatrix-seed`; observe that skill reads README/manifests without asking; verify 2–4 entries proposed; approve batch; call `context_status` and verify entry count increased | PENDING |
| AC-07 | `/unimatrix-seed` Level 1+ requires explicit human opt-in per exploration category; presents a menu and waits for selection before proceeding | manual | Run `/unimatrix-seed` through Level 0 completion; verify skill displays a menu of categories (module structure, conventions, build/test) and stops; verify no Level 1 file reads occur until selection is provided | PENDING |
| AC-08 | Level 0 uses batch approval by default; Level 1+ uses individual entry approval; only approved entries are stored | manual | Run full seed; at Level 0 reject entire batch; at Level 1 reject one specific entry; verify rejected batch entries and rejected Level 1 entry are absent from `context_search` results | PENDING |
| AC-09 | `/unimatrix-seed` terminates after 2 opt-in levels; no Level 3 offered | manual | Run through Level 0 → Level 1 → Level 2; verify at GATE_2 only a DONE summary is shown; no "continue to Level 3?" prompt appears | PENDING |
| AC-10 | Both skills delivered as `.claude/skills/unimatrix-init/SKILL.md` and `.claude/skills/unimatrix-seed/SKILL.md` with YAML frontmatter (`name`, `description`) | grep | Verify files exist at exact paths; check frontmatter: `grep -E "^name:|^description:" .claude/skills/unimatrix-init/SKILL.md .claude/skills/unimatrix-seed/SKILL.md` | PENDING |
| AC-11 | The CLAUDE.md block is self-contained — a developer with no Unimatrix knowledge can read it and understand available skills and when to use them | manual | Peer review: ask a developer unfamiliar with Unimatrix to read the appended block and answer "what skills exist and when do I use each?" without consulting other docs | PENDING |
| AC-12 | `/unimatrix-init` SKILL.md includes a disambiguation notice distinguishing it from the `uni-init` agent | grep | `grep -i "uni-init" .claude/skills/unimatrix-init/SKILL.md` — verify the file mentions both tools and distinguishes their purpose | PENDING |
| AC-13 | `/unimatrix-seed` warns if seed entries already exist (via `context_search` pre-check) and offers to supplement rather than re-seed, before any Level 0 stores | manual | Pre-populate knowledge base with ≥3 entries in `convention`/`pattern`/`procedure`; run `/unimatrix-seed`; verify warning appears before any new entry proposals are made | PENDING |
| AC-14 | The sentinel marker includes a version number: `<!-- unimatrix-init v1: DO NOT REMOVE THIS LINE -->` | grep | `grep "unimatrix-init v1" .claude/skills/unimatrix-init/SKILL.md` — verify sentinel string is present in SKILL.md instruction; also verify in generated CLAUDE.md after running the skill | PENDING |
