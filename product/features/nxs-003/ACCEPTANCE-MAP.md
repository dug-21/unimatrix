# nxs-003 Acceptance Criteria Map

| AC-ID | Description | Verification Method | Verification Detail | Status |
|-------|-------------|--------------------|--------------------|--------|
| AC-01 | `unimatrix-embed` library crate compiles within the Cargo workspace with `cargo build` | shell | `cargo build --workspace` succeeds with zero errors. Verify `Cargo.toml` at workspace root includes `unimatrix-embed` as a member. Crate is edition 2024. | PENDING |
| AC-02 | `OnnxProvider::new(config)` loads the ONNX model and tokenizer, downloading from HuggingFace Hub on first use to `~/.cache/unimatrix/models/` | test | Integration test creates `OnnxProvider` with default config. On first run, model files appear in cache directory. On second run, no download occurs (file modification times unchanged). Assert provider is ready for inference. | PENDING |
| AC-03 | `OnnxProvider::embed(text)` returns a `Vec<f32>` with exactly 384 elements for any non-empty input string | test | Integration test embeds three distinct strings ("hello world", a long paragraph, a single word). Each result has `.len() == 384`. All elements are finite (no NaN, no infinity). | PENDING |
| AC-04 | `OnnxProvider::embed_batch(texts)` returns one 384-d embedding per input text. Output count equals input count. | test | Integration test embeds a batch of 5 texts. Result has `.len() == 5`. Each inner vec has `.len() == 384`. | PENDING |
| AC-05 | All output embeddings are L2-normalized: L2 norm within tolerance of 1.0 (\|norm - 1.0\| < 0.001) | test | Integration test embeds 10 varied texts (short, long, empty, unicode). For each embedding, compute L2 norm and assert `(norm - 1.0).abs() < 0.001`. | PENDING |
| AC-06 | `prepare_text(title, content, ": ")` concatenates as `"{title}: {content}"`. Empty title → content only. Empty content → title only. | test | Unit tests: `prepare_text("JWT", "Validate exp", ": ")` == `"JWT: Validate exp"`, `prepare_text("", "content only", ": ")` == `"content only"`, `prepare_text("title only", "", ": ")` == `"title only"`, `prepare_text("", "", ": ")` == `""`. | PENDING |
| AC-07 | `embed_entry(provider, title, content)` returns a single 384-d normalized embedding for concatenated title+content | test | Integration test: `embed_entry(provider, "Auth", "Use JWT")` returns 384-d embedding. Assert dimension and normalization. Assert result equals `provider.embed(&prepare_text("Auth", "Use JWT", ": "))`. | PENDING |
| AC-08 | Semantically similar texts have high cosine similarity (> 0.7). Unrelated texts have low cosine similarity (< 0.3). | test | Integration test with known pairs: similar ("Rust error handling best practices", "How to handle errors in Rust") → similarity > 0.7; dissimilar ("Rust error handling", "Recipe for chocolate cake") → similarity < 0.3. Computed via `cosine_similarity` helper. | PENDING |
| AC-09 | `EmbeddingProvider` trait is object-safe: usable as `&dyn EmbeddingProvider` or `Box<dyn EmbeddingProvider>` | test | Compile-time test: function accepts `&dyn EmbeddingProvider` and calls all four trait methods. Second function accepts `Box<dyn EmbeddingProvider>`. Both compile without error. | PENDING |
| AC-10 | `OnnxProvider` is `Send + Sync`, shareable via `Arc<OnnxProvider>` | test | Compile-time assert: `fn assert_send_sync<T: Send + Sync>() {}; assert_send_sync::<OnnxProvider>();`. Integration test: wrap `OnnxProvider` in `Arc`, clone to two threads, embed from both, assert both get valid results. | PENDING |
| AC-11 | Batch embedding of N texts produces same embeddings (within fp tolerance) as embedding each individually | test | Integration test: embed 5 texts individually via `embed()`, then batch via `embed_batch()`. For each pair, assert `(individual[i] - batch[i]).abs() < 1e-5` element-wise. | PENDING |
| AC-12 | Empty string input returns an embedding (not an error) — model produces valid vector for empty input | test | Integration test: `provider.embed("")` returns `Ok(embedding)` with 384 elements, L2 norm within tolerance of 1.0. | PENDING |
| AC-13 | `EmbedConfig` supports configuring: model selection, cache directory, batch size, text separator | test | Unit test: create `EmbedConfig` with non-default values for all four fields, assert each field set correctly. Integration test: create `OnnxProvider` with custom `cache_dir` (temp directory); model files appear in custom directory. | PENDING |
| AC-14 | All public API functions return typed `Result` errors (no panics). Error types cover model loading, inference, tokenization, and download failures. | test | Unit tests construct each `EmbedError` variant and verify `Display` output. Integration test: `OnnxProvider::new()` with non-existent read-only cache path returns `EmbedError::Io`. Corrupt model file returns model loading error. | PENDING |
| AC-15 | `#![forbid(unsafe_code)]` at crate level | grep | `grep -n 'forbid(unsafe_code)' crates/unimatrix-embed/src/lib.rs` returns a match. Compiler enforces — any unsafe block causes compile error. | PENDING |
| AC-16 | `OnnxProvider::dimension()` returns 384 for all catalog models | test | Unit test: for each `EmbeddingModel` variant, assert `model.dimension() == 384`. Integration test: assert `provider.dimension() == 384`. | PENDING |
| AC-17 | `EmbeddingModel` enum provides all 7 pre-configured 384-d models with HuggingFace ID, dimension, max seq length, ONNX filename | test | Unit test iterates all 7 variants, asserts each has non-empty `model_id()`, `dimension() == 384`, `max_seq_length()` is 256 or 512, `onnx_filename()` is non-empty. | PENDING |
| AC-18 | Mean pooling correctly applies attention mask — masked (padding) tokens do not contribute to pooled embedding | test | Unit test with hand-crafted example: token embeddings `[[1.0, 2.0], [3.0, 4.0], [0.0, 0.0]]`, attention mask `[1, 1, 0]`, expected `[2.0, 3.0]`. Assert output matches expected. | PENDING |
| AC-19 | Test infrastructure: cosine_similarity helper, dimension/normalization assertion helpers, MockProvider for downstream testing | test | Unit test: `MockProvider` implements `EmbeddingProvider`, `embed()` returns 384-d normalized vector, `embed_batch()` returns one per text. `cosine_similarity` correct for known vectors. `assert_dimension` and `assert_normalized` pass for valid, panic for invalid. | PENDING |
