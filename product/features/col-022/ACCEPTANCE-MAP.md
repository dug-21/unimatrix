# col-022 Acceptance Criteria Map

| AC-ID | Description | Verification Method | Verification Detail | Status |
|-------|-------------|--------------------|--------------------|--------|
| AC-01 | `context_cycle` MCP tool registered with params: `type` (required, "start"/"stop"), `topic` (required), `keywords` (optional, up to 5 strings) | test | Unit test: tool schema introspection verifies params and types; integration test: tool call roundtrip returns valid response |
| AC-02 | `context_cycle(type: "start", topic: "X")` causes session's `feature_cycle` to be set to "X" in SessionRegistry and SQLite sessions table | test | Integration test: call context_cycle(start), query SessionRegistry for feature_cycle match, query SQLite sessions table row for feature_cycle = "X" |
| AC-03 | Explicit `context_cycle(type: "start")` is authoritative -- overrides any prior heuristic attribution via `set_feature_force` | test | Integration test: set feature via eager attribution, call context_cycle(start) with different topic, verify SessionRegistry and SQLite both reflect the new explicit value; verify `SetFeatureResult::Overridden` returned |
| AC-04 | `context_cycle(type: "stop", topic: "X")` records a cycle-end observation event with session_id and topic | test | Integration test: call context_cycle(stop), query observations table for event_type = "cycle_stop" with matching session_id and topic |
| AC-05 | `context_cycle` returns "noted" acknowledgment; does not confirm attribution outcome | test | Unit test: parse response text, verify contains "noted" or equivalent acknowledgment; verify no `was_set` field in response |
| AC-06 | `topic` validated: non-empty, max 128 chars, no control characters (via `sanitize_metadata_field`) | test | Unit test on `validate_cycle_params`: empty string rejected, 129-char string rejected, control chars rejected, 128-char string accepted |
| AC-07 | `type` only accepts "start" or "stop"; any other value returns validation error | test | Unit test on `validate_cycle_params`: "pause" rejected, "restart" rejected, "" rejected, "Start" rejected (case-sensitive), "start" and "stop" accepted |
| AC-08 | Existing heuristic attribution (eager voting, majority vote) unchanged when `context_cycle` not called | test | Integration test: session without cycle call, send topic signals, verify eager attribution resolves at 3+ count / >60% share; verify majority vote on close |
| AC-09 | After `context_cycle(start)`, eager attribution does not overwrite the explicit value (`set_feature_if_absent` is no-op) | test | Integration test: call context_cycle(start) to set feature, send topic signals for different feature, verify feature_cycle unchanged |
| AC-10 | Hook handler processes `context_cycle` PreToolUse within 50ms latency budget | test | Integration test: measure end-to-end hook latency for PreToolUse with context_cycle tool_input, assert < 50ms |
| AC-11 | `context_retrospective(feature_cycle: "X")` finds observations for sessions where `context_cycle(start, "X")` was called | test | Integration test: create session, call context_cycle(start, "X"), record observations, run context_retrospective(feature_cycle: "X"), verify observations returned |
| AC-12 | Wire protocol backward compatible: old hook binaries do not crash on new event types | test | Unit test: deserialize unknown event_type string into ImplantEvent, verify no panic and event processed through generic path |
| AC-13 | `keywords` accepts array of up to 5 strings, each max 64 chars; excess entries truncated to 5, long strings truncated to 64 | test | Unit test on `validate_cycle_params`: 7-element array truncated to 5; 65-char string truncated to 64; empty array accepted; 5x64-char strings accepted |
| AC-14 | Keywords stored in session record (SQLite sessions table) alongside feature_cycle, retrievable for future injection use | test | Integration test: call context_cycle(start) with keywords, query SQLite sessions table, deserialize keywords column as JSON array, verify round-trip fidelity |
| AC-15 | Keywords pass through hook/UDS path and persist via fire-and-forget pattern | test | Integration test: full hook-to-UDS-to-SQLite roundtrip with keywords in PreToolUse tool_input; verify keywords arrive in sessions table |
