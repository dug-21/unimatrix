# crt-020 Acceptance Criteria Map

| AC-ID | Description | Verification Method | Verification Detail | Status |
|-------|-------------|--------------------|--------------------|--------|
| AC-01 | Schema migration v12→v13 adds `implicit_votes_applied INTEGER NOT NULL DEFAULT 0` to `sessions`. All existing sessions in a v12 database have `implicit_votes_applied = 0` after migration. | test | `cargo test -p unimatrix-store test_migration_v12_to_v13` — assert column exists, all rows default to 0, schema version = 13 | PENDING |
| AC-02 | Each background maintenance tick queries sessions where `implicit_votes_applied = 0 AND status = 1 AND outcome IS NOT NULL`, up to `IMPLICIT_VOTE_BATCH_LIMIT` (default 500) sessions per tick. | test | `cargo test -p unimatrix-server test_implicit_vote_sweep_batch_cap` — seed 600 eligible sessions, run one tick, assert exactly 500 processed | PENDING |
| AC-03 | For a Completed session with `outcome = "success"` and 3 distinct entries in `injection_log`: after the implicit vote sweep, each of the 3 entries has `helpful_count` incremented by exactly 1, and `implicit_votes_applied = 1`. | test | `cargo test -p unimatrix-server test_success_session_votes_applied` — assert per-entry `helpful_count` delta = 1, assert session flag = 1 | PENDING |
| AC-04 | Sessions with `outcome != "success"` (rework, abandoned, NULL) and sessions with `status = TimedOut (2)` or `status = Abandoned (3)` produce zero signal. `unhelpful_count` is not modified by this feature under any circumstances. | test | `cargo test -p unimatrix-server test_zero_signal_sessions` — (a) rework: no vote increment; (b) TimedOut: no vote increment; (c) NULL outcome: excluded from sweep query; (d) assert `unhelpful_count` unchanged for all cases | PENDING |
| AC-05 | After processing, `implicit_votes_applied = 1`. A session is never processed twice. Running the sweep twice does not change `helpful_count` after the second run. | test | `cargo test -p unimatrix-server test_implicit_vote_idempotency` — run sweep twice; assert `helpful_count` unchanged after second invocation | PENDING |
| AC-06 | A Completed session with `outcome = "success"` and 2 `injection_log` rows for the same `entry_id` results in exactly 1 `helpful_count` increment for that entry (per-session per-entry deduplication via HashSet). | test | `cargo test -p unimatrix-store test_duplicate_injection_log_dedup` — seed session with duplicate entry rows; assert `helpful_count` delta = 1, not 2 | PENDING |
| AC-07 | Confidence is recomputed for each entry that receives an implicit vote, using the current Bayesian prior parameters `(alpha0, beta0)` from `ConfidenceStateHandle`. The recomputed confidence is persisted before the transaction commits. | test | `cargo test -p unimatrix-server test_implicit_vote_confidence_recomputed` — read confidence before sweep, run sweep, assert confidence value changed and matches expected Bayesian computation | PENDING |
| AC-08 | A session closed by the Stop hook in real-time has `implicit_votes_applied = 1` set at session close time. The next background tick excludes this session from the implicit vote sweep; `helpful_count` does not change after the tick. | test | `cargo test -p unimatrix-server test_stop_hook_flag_prevents_double_count` — simulate Stop hook close, assert flag=1, run tick, assert `helpful_count` unchanged | PENDING |
| AC-09 | The background tick implicit vote sweep and the Stop hook real-time path are disjoint. A session with `implicit_votes_applied = 1` (Stop hook path) is not re-processed by the sweep. A session with `implicit_votes_applied = 0` (offline session) IS processed by the sweep. | test | `cargo test -p unimatrix-server test_disjoint_processing_paths` — two sessions (one flagged by Stop hook, one offline); assert only the offline session gets votes after tick | PENDING |
| AC-10 | Given 1,000 historical Completed sessions with `outcome = "success"` and `implicit_votes_applied = 0`: the first background tick processes exactly `IMPLICIT_VOTE_BATCH_LIMIT` sessions (oldest-first); subsequent ticks process the remainder until all sessions have `implicit_votes_applied = 1`. Each batch completes without exceeding the tick timeout. | test | `cargo test -p unimatrix-server test_cold_start_batch_ordering` — seed 1,000 sessions with known `ended_at` ordering; after tick 1 assert exactly 500 processed and those 500 are the oldest; after tick 2 assert remaining 500 processed | PENDING |
| AC-11 | A Completed session with `outcome = "success"` and 0 rows in `injection_log` (no injections): `implicit_votes_applied` is set to 1 and no vote counters are modified (no-op path, no error). | test | `cargo test -p unimatrix-server test_zero_injection_success_session` — run sweep on session with empty `injection_log`; assert flag=1, no `helpful_count` changes, no error | PENDING |
