# Component D — mcp/response/mod.rs

## Purpose

Test module for status report formatting. Contains test helper functions that construct
`StatusReport` literals and inline fixture tests. After crt-048 all 16 field references
to the two removed fields must be deleted and 4 tests that assert on those fields must be
deleted entirely. This component's changes are purely mechanical field removals and test
deletions — no logic changes.

All removals cause compile errors if missed (R-02). The build gate is the primary
detection mechanism. No partial removal can reach runtime.

and any blank lines separating it from adjacent tests.

### 1. test_coherence_json_all_fields (lines ~1474-1533)

### 2. test_coherence_json_f64_precision (lines ~1535-1550)

Reason: Sets `confidence_freshness_score` values and asserts JSON contains them.
Also references `make_coherence_status_report()` which will have those fields removed.

Delete: The entire function from `// UT-C6-02:` comment through the closing `}`.

Note: The f64 precision behavior for the remaining coherence fields (e.g. `coherence`
itself at 0.845) is still implicitly tested by `test_coherence_markdown_section` and
similar tests that call `make_coherence_status_report()`. If a dedicated f64 precision
test for the surviving fields is desired, it is a new test — not in scope for crt-048.

### 3. test_coherence_stale_count_rendering (lines ~1714-1738)

Reason: Calls `make_coherence_status_report()` and asserts presence of "Stale confidence:
15 entries" and "Stale confidence entries: 15" in Summary and Markdown formats. Both
rendering branches that produced these strings are deleted. Also contains line 1731
(`report2.stale_confidence_count = 0`) — a field mutation that would not compile.

Delete: The entire function from the test comment through line 1738 closing `}`.

### 4. test_coherence_default_values (lines ~1789-1803)

Reason: Asserts `report.confidence_freshness_score == 1.0` (line 1794) and
`report.stale_confidence_count == 0` (line 1798). Fields no longer exist.

`embedding_consistency_score`, `contradiction_density_score`, `confidence_refreshed_count`,
`graph_stale_ratio`, `graph_compacted`, `maintenance_recommendations`) are about surviving
fields and could be retained in a renamed test. However, since the existing test function
is named `test_coherence_default_values` and tests both surviving and deleted fields
together, the cleanest approach is to delete it entirely. Any agent that wants default-field
coverage of surviving fields can add a new test — that is out of scope for crt-048.

Delete: The entire function from `// UT-C6-11:` comment through line 1803 closing `}`.
