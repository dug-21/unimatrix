# Pseudocode: C8 Compaction Integration

## Purpose

Wire graph compaction into the context_status handler. When maintain=true and stale ratio exceeds threshold, re-embed active entries and compact the HNSW index.

## Files Modified

- `crates/unimatrix-server/src/tools.rs` -- context_status handler

## Integration Point

Added after confidence refresh (C5) and co-access cleanup, before lambda computation. Only runs when maintain=true.

## Pseudocode

```
// Inside context_status handler, after confidence refresh:

let graph_compacted: bool;
let graph_stale_ratio: f64;

// Compute stale ratio
let point_count = vector_index.point_count();
let stale_count_hnsw = vector_index.stale_count();
graph_stale_ratio = if point_count == 0 { 0.0 }
                    else { stale_count_hnsw as f64 / point_count as f64 };

if maintain_enabled && graph_stale_ratio > DEFAULT_STALE_RATIO_TRIGGER:
    // Check embed service availability (SR-04, R-09)
    match embed_handle.get_adapter().await:
        Ok(adapter):
            // Gather active entries for re-embedding
            // all_active_entries is already available from the entry scan
            let pairs: Vec<(String, String)> = all_active_entries.iter()
                .map(|e| (e.title.clone(), e.content.clone()))
                .collect()

            match adapter.embed_entries(&pairs):
                Ok(embeddings):
                    // Build (entry_id, embedding) pairs
                    let compact_input: Vec<(u64, Vec<f32>)> = all_active_entries.iter()
                        .zip(embeddings.into_iter())
                        .map(|(entry, emb)| (entry.id, emb))
                        .collect()

                    match vector_index.compact(compact_input):
                        Ok(()) =>
                            graph_compacted = true
                        Err(e) =>
                            warn!("graph compaction failed: {}", e)
                            graph_compacted = false
                            // Add recommendation about failed compaction
                Err(e) =>
                    warn!("re-embedding for compaction failed: {}", e)
                    graph_compacted = false
        Err(_) | not available:
            // Embed service unavailable -- graceful skip
            graph_compacted = false
            // Recommendation will be generated by generate_recommendations
            // if stale ratio is high enough to affect lambda
else:
    graph_compacted = false
```

## Embed Service Availability Check

The embed service is lazily loaded. The current pattern for checking availability:

```
// Check if embed_handle has an adapter ready
// If get_adapter() returns None or Err, compaction is skipped
```

Check the existing embed_handle usage in context_status for the exact pattern. The crt-005 compaction uses the same gating.

## Error Handling

- **Embed service unavailable**: compaction skipped, graph_compacted=false. The stale ratio is still reported. Recommendation generation (C4) will produce a recommendation if lambda is below threshold.
- **Re-embedding fails**: compaction skipped, log warning.
- **compact() fails**: old graph untouched (build-new-then-swap per ADR-004). graph_compacted=false.
- **All failures are non-fatal**: context_status always completes with whatever coherence data is available.

## Post-Compaction State

After successful compaction:
- vector_index.stale_count() == 0
- vector_index.point_count() == active entry count
- VECTOR_MAP entries match new data_ids
- Search results remain consistent (same entry_ids, possibly different ordering due to HNSW non-determinism)

## Key Test Scenarios

1. maintain=true + stale ratio > 10%: compaction triggers (AC-11)
2. maintain=false + high stale ratio: compaction skipped (R-07)
3. Embed service unavailable: compaction skipped gracefully (R-09, AC-11)
4. Post-compaction stale_count == 0 (AC-13)
5. Post-compaction search returns same entries (AC-20)
6. graph_compacted=true in report when compaction ran (AC-14)
7. graph_stale_ratio reported even when compaction not run
8. Empty knowledge base: no compaction (no stale nodes) (R-18)
